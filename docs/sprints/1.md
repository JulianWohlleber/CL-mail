# Sprint 1 — Visual polish + a11y audit

**Lead:** Felix (visual), Lena (system)
**Tag:** v0.1.1

## Research

**Felix:** "We're shipping focus rings only on `<button>` elements that
inherit the user-agent default — they look fine in light mode but disappear
on the dark-sunken sidebar. iA Writer's principle is *the type should be
the focus*; instead I want a 1px accent outline that doesn't shift layout."

**Lena:** "Nine places set `font-size: 11px` inline. Nine. We need a tiny
typographic scale and stop hand-writing pixel values. Also: the thread row
border-bottom is `var(--border)` (full opacity) — that's heavier than the
sidebar separator. Both should read the same."

## Design

- New token `--accent-ring` (accent at 35% opacity) for visible-but-quiet focus
- Add a typographic scale (`--text-2xs/xs/sm/base/lg`) so we stop inlining sizes
- Standardize border tokens: `--border` for separators, `--border-strong` for
  inputs/cards under focus
- Animated transitions ≤ 160 ms — anything longer feels app-like, not paper-like

## Development

- `globals.css`: add `--accent-ring`, `--text-*` scale, focus-visible style for
  all interactive elements
- ThreadItem: drop hand-rolled font-size, use the scale
- ReadingPane action-button row: equal hit-target, visible focus
- Subtle motion: 120 ms ease-out on sidebar-item highlight

## Testing

- Regression test #12: ensure `--accent-ring` token is defined (so a future
  CSS-cleanup PR can't kill keyboard accessibility for free)
- Manual: tab through thread list → search bar → compose modal; ring visible
  in light + dark + system
