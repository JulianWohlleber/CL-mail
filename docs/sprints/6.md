# Sprint 6 — Search refinement: `from:` / `has:` filter operators

**Lead:** Sam (QA), David (backend)
**Tag:** v0.1.6

## Research

**Sam:** "Three of five test users tried `from:hannes`. Then `subject:invoice`.
Our search just stuffs the whole string into FTS5 MATCH and returns nonsense
because the colons throw off the parser. We're discoverable enough that
users *expect* operators."

**David:** "FTS5 already supports column-restricted matches with `from_address:term`
syntax — we just need to translate user-friendly `from:` / `subject:` / `has:`
operators into the right FTS5 expression."

## Design

Recognized operators:

```
from:foo        — restrict to messages whose from_address contains foo
subject:invoice — restrict to messages whose subject contains invoice
has:attachment  — only threads with attachments
has:unread      — only unread
is:unread       — alias of has:unread
```

Free text outside operators searches body + subject as before.

Operators co-exist with free text: `from:hannes invoice` works.
