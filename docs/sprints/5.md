# Sprint 5 — Compose UX + draft autosave

**Lead:** Priya (frontend), Sam (QA)
**Tag:** v0.1.5

## Research

**Sam:** "Compose has zero recovery story. Hit Cmd+W mid-message → it's
gone. Twice already in QA. The user has no way to tell whether their
message even started saving."

**Priya:** "The compose window has body/subject/recipients in component
state. They never touch the store, never touch disk. We need a debounced
save to a local-drafts table, replayed on next open."

## Design

- New `drafts` table (id, accountId, to/cc/bcc, subject, body, updated_at)
- ComposeWindow autosaves every 1.2 s of idle typing
- Status pill in the compose footer: "Saved · 23 s ago"
- Restore-on-open: if there's a saved draft for the same account+mode+threadId,
  open with it

## Development

- DB migration #011
- IPC: drafts list / save / delete
- ComposeWindow debounced effect on body/subject/recipients
- Reuse same id across saves; delete on successful send or discard
