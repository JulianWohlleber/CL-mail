# Sprint 2 — First-run onboarding

**Lead:** Lena (system), Maya (eng lead)
**Tag:** v0.1.2

## Research

**Lena:** "Watched three people open the app for the first time. None of
them noticed the keyboard shortcuts at all — they all tried clicking
buttons that don't exist in our minimal chrome. The WelcomeScreen lists
features but doesn't teach a single shortcut. We need an inline coachmark
that fires after the first thread is selected."

**Maya:** "And our 'Tutorial' component is mounted but never given a real
script. Let's just remove the dead Tutorial widget and put one well-placed
hint where the user is already looking — the reading pane."

## Design

- Drop the noisy multi-step Tutorial modal
- Add a single dismissable hint card in the reading pane on first thread
  selection: "Press `j`/`k` to navigate, `e` to archive, `?` for all shortcuts"
- Persist a `firstRunHintDismissed` flag in settings

## Development

- New `FirstRunHint` component, mounted next to the reading pane title
- Settings flag, dismiss on close
- Drop the unused Tutorial wiring (component left untouched in case we
  want to reuse it for migration tooltips)

## Testing

- Regression test #13: the hint reads from settings (so it can't be made
  permanent by a future refactor)
